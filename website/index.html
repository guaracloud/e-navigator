Capabilities

Runtime signals without application SDKs.

The current foundation captures process, network, DNS, and HTTP/1 and HTTP/2 traffic at runtime, reassembles and matches Kafka, MongoDB, MySQL, NATS, PostgreSQL, Redis, and gRPC requests to their responses for real latency and status, reads TLS plaintext at the userspace library boundary through OpenSSL and GnuTLS uprobes (interception, not on-the-wire decryption), accounts for TCP retransmits, resets, and state transitions, samples and symbolizes CPU profiles, and turns resource, trace, and security observations into bounded native E-Navigator signals. An optional capture filter scopes which workloads are probed by Kubernetes namespace and label.

Pipeline

Source -> Processor -> Generator -> Sink

Static module registration keeps runtime behavior inspectable and testable.

Attribution

Host, process, container, Kubernetes

Signals carry context when evidence exists and warnings when it does not.

Export

JSON, Prometheus, OTLP, pprof

JSON stdout is the default. Prometheus, OTLP, and a local pprof profile endpoint are opt-in, bounded, and family-selectable.

Scope control

Namespace and label capture filter

Opt-in cgroup-id filtering probes only the workloads you choose; excluded workloads pay near-zero per-syscall cost.

Boundaries

No hidden production claims

Storage, UI, runtime pprof upload, broad collector compatibility, reduced privilege, and reduced overhead stay non-claims until directly proven.

Proof status

Public evidence is curated, not dumped.

E-Navigator separates local tests, Docker smoke, Helm rendering, and privileged runtime proof. The proof report is the current public evidence map.

Proven locally

Static runtime, strict config schema validation, config selectors, envelopes, DNS configurable source limits, HTTP configurable parser limits and request/status coverage, gRPC metadata/status, profile fixture normalization filtering, Kafka request/ApiVersions-error, MongoDB command/error, MySQL command/error, NATS command/error, PostgreSQL query/error, and Redis command/error parser coverage, flow-attribution warnings, synthetic protocol request/error spans, the Kubernetes-aware capture filter (policy, resolver, map diff, cgroup-path parsing, glob and cgroup-path fuzz targets, a privacy invariant test, and rule-eval/map-diff benches), Docker smoke, and chart rendering.

Runtime-proven slices

Selected exec, network, DNS, HTTP, profile, resource, Prometheus, OTLP, and seccomp paths, plus homelab request/response matching for Redis, PostgreSQL, MongoDB, NATS, gRPC, and Kafka. Local OrbStack proof covers outbound and inbound HTTP capture, HTTP/2 with HPACK matching, deeper Redis reassembly and TLS slices, native and CPython profile symbolization, a served pprof endpoint, TCP reset and state-transition observation, and Kubernetes-aware capture filtering. These are bounded runtime slices, not production coverage.

Still partial

Native flow-byte live export, symmetric all-node DNS/HTTP capture, live MySQL capture proof, broad protocol and TLS coverage, native DWARF pod-process coverage under high node load, live TCP retransmit induction, production backend proof, production soak, and reduced privilege.

Install paths

Run locally, then verify releases before production.

Synthetic local run

Exercise the pipeline without privileged Linux or Kubernetes dependencies.

cargo run --locked -p e-navigator-cli -- --source synthetic

Helm OCI chart

Install the published chart, then pin digest-backed images after verification.

helm upgrade --install e-navigator oci://ghcr.io/guaracloud/charts/e-navigator \
  --version 0.1.2 \
  --namespace e-navigator-system \
  --create-namespace

Verify release artifacts

Check release manifest, checksums, Cosign signatures, SBOMs, image digest, and chart digest.

gh release download v0.1.2 --repo guaracloud/e-navigator --dir e-navigator-release
jq -r '.images[0].reference' e-navigator-release/release-manifest.json

# Capabilities

This document is the current public capability map for E-Navigator. It is
intentionally shorter than the internal proof trail and should be updated only
when implementation and evidence change together.

## Capability Inventory

E-Navigator is a node-local Rust/eBPF signal plane with a statically registered
`Source -> Processor -> Generator -> Sink` runtime. The CLI starts a unified
runtime by default: enabled Aya capture sources, host resource collection, and
opt-in CPU profiling execute concurrently in one process. Legacy Aya modes are
kept for focused diagnostics, and the synthetic fixture remains available for
non-privileged testing. Runtime config validates known module names, duplicate
modules, bounded queues, source-supervisor policy and shutdown bounds, derived
signal fan-out, source limits, generator limits, and sink settings before the
runner starts. On Linux 5.8 or newer, a positive kernel feature probe selects
BPF ring buffers for kernel-to-userspace event delivery. Older kernels use a
separately built perf-event object only when the probe positively reports that
ring-buffer maps are unsupported. For the network source's scalar read/write
byte accounting, a second strict preflight selects BTF-backed fexit hooks on
compatible kernels and retains syscall tracepoints as the compatibility path.

### Kernel Event Transport

| Control | Default | Capability |
| --- | --- | --- |
| `[ebpf]` | `event_transport = "auto"`, `ring_buffer_bytes = 262144` | Selects one of two embedded eBPF objects before load: shared RingBuf event maps on supported kernels or per-CPU perf-event arrays on kernels that positively report RingBuf unsupported. Forced `ring_buffer` and `perf_buffer` modes support strict diagnostics and A/B runs. Ring capacity is a power of two from 4 KiB through 16 MiB and must match the runtime page size. Native per-source metrics identify the active transport and account for aggregate loss, perf lost records, and RingBuf producer reservation failures. Probe errors fail source startup rather than silently falling back. |
| `[ebpf]` network I/O hook | `network_io_hook = "auto"` | Selects BTF-backed `ksys_read` and `ksys_write` fexit only after tracing-program, kernel-BTF, and target-function preflight succeeds. A positively unsupported tracing type, absent BTF file, or absent target uses the four retained syscall tracepoints with a warning. Indeterminate probe, permission, malformed-BTF, verifier, load, and attach failures stop source startup. Forced `fexit` and `tracepoint` modes support strict diagnostics and A/B runs. |

### Source Capabilities

| Source | Default | Capability |
| --- | --- | --- |
| `source.aya_exec` | Enabled in unified mode | Captures Linux exec success and process exit events from Aya/eBPF, with bounded optional argv capture, uid, pid, cgroup id, executable/command, exit timestamp, and best-effort container context from procfs. |
| `source.aya_network` | Enabled in unified mode | Captures TCP IPv4/IPv6 connection open, close, and failure observations, including pid/uid/command, fd, local/remote address and port, duration, errno, and optional sent/received byte counters, plus TCP retransmit, reset (send/receive), and state-transition observations. Scalar `read(2)`/`write(2)` byte accounting uses preflighted BTF fexit on compatible kernels and syscall tracepoints otherwise; the other network hooks remain tracepoint based. |
| `source.aya_dns` | Opt-in | Captures bounded DNS query/response packets over UDP or TCP, parses A, AAAA, CNAME, and other query types, response code, server address/port, latency when present, and process/container context. |
| `source.aya_http` | Opt-in | Captures bounded cleartext outbound HTTP request bytes from write-style syscall paths and, when inbound capture is enabled, server-side HTTP request bytes from accept-tracked connections and read-style syscall paths. Parses request method/target metadata and W3C trace context when present and emits protocol request observations tagged with the client or server capture role. |
| `source.aya_protocol` | Opt-in | Captures bounded socket payload windows (configurable `capture_bytes_per_syscall`, default 1 KiB, up to 4 KiB per syscall as spliced 256-byte segments) in both directions for outbound TCP connections to configured HTTP/1, HTTP/2 (h2c/gRPC), WebSocket, gRPC-Web, Kafka, MongoDB, MySQL, NATS, PostgreSQL, and Redis ports. Optional `inbound_enabled` capture tracks accepted server sockets, resolves configured local ports from bounded bind/accept state or the observed process's bounded procfs socket table, and preserves client/server capture roles. The source reassembles per-connection streams with explicit truncation/desync accounting, matches responses to a bounded in-flight request queue (stream-id keyed for HTTP/2), and emits protocol request observations through bounded parsers. HTTP/2 HEADERS and same-stream CONTINUATION sequences are reassembled before connection-scoped HPACK decoding. Valid extension-free HTTP/1 upgrades switch a connection to metadata-only RFC 6455 frame parsing, with the kernel connection generation preventing state reuse across recycled file descriptors. Binary and base64-text gRPC-Web envelopes produce bounded RPC metadata and trailer status without protobuf or application payload export. Native counters account for WebSocket upgrades, frames, rejected transitions, and gRPC-Web requests. HTTP/3 and generic QUIC semantics are an explicit non-claim because the current source observes TCP payload windows, while QUIC uses UDP and encrypts its HTTP semantics. |
| `source.aya_tls` | Opt-in | Captures TLS plaintext at supported userspace library boundaries (interception, not on-the-wire decryption). Dynamic support covers OpenSSL 1.1.1/3 with the complete classic and `_ex` read/write plus fd-association export set and GnuTLS ABI 30 with the standard integer socket transport. Static Go support covers unstripped Linux/amd64 ELF executables built with Go 1.24 through 1.26: bounded procfs discovery requires exact build info and static `crypto/tls.(*Conn).Read`/`Write` plus `net.(*netFD).Read`/`Write` symbols, fully decodes bounded method bodies, transactionally attaches every decoded return site, correlates by goroutine rather than OS thread, and publishes only an audited version-specific fd offset. OpenSSL/GnuTLS and Go plaintext reuse connect/accept tuples and the existing bounded multi-segment reassembler, parsers, and matcher under `source.aya_tls` provenance. Candidates are rescanned every 15 seconds. Unknown versions, incomplete exports/symbols, stripped Go binaries, unsupported Go architectures, custom BIO/transports, BoringSSL, rustls, and statically bundled Node/JVM TLS fail closed. Native bounded metrics expose target coverage plus Go entries/exits, layout and pending misses, state replacements/update failures, fd resolution, and output attempts. |
| `source.aya_cpu_profile` | Opt-in config in unified mode | Samples periodic on-CPU profiles through Aya perf events and can separately enable scheduler-based off-CPU and futex-wait lock profiles. Event-driven modes use distinct fixed 4,096-entry pending maps, positive duration thresholds, per-CPU rate caps, process-exit cleanup, fail-closed scheduler-layout and architecture preflight, and native input/update/replacement/stack-failure/below-minimum/rate/output counters. Duration travels as `profiling.sample.weight_nanos` through the existing bounded session, pprof, and OTLP Profiles paths. All modes share configurable stack depth (`max_frames_per_sample`, default 64, up to 128 frames; the frame-pointer path is additionally bounded by `kernel.perf_event_max_stack`, 127 by default), explicit capped-depth and backpressure accounting, capture filtering, and pid-namespace translation with identity-verified fallback. Native unwind uses bounded DWARF/CFI rules parsed from `.eh_frame`, with frame pointers as fallback. Exact CPython 3.11 and 3.12 layouts produce leaf-first function/file/line frames from bounded `/proc/<pid>/mem` reads. Native instruction pointers resolve through procfs maps, local ELF symbols, and, when a workload already publishes one, a bounded target-mount `/tmp/perf-<pid>.map` for Node/V8 or JVM generated-code names. E-Navigator does not enable JIT flags or agents, and a symbol map does not guarantee unwind through opaque JIT frames. |
| `source.host_resource` | Enabled in unified mode | Samples procfs, sysfs, and cgroup files for node CPU/load/memory/filesystem/disk I/O, process resource usage, cgroup CPU/memory/pids/file-descriptor observations, and bounded warning aggregation. |
| `source.synthetic_exec` | Enabled only in synthetic mode | Emits synthetic exec, network, DNS, HTTP/request, trace, profile, and resource fixtures for non-privileged local exercise. |

### Deployment Privilege Profiles

| Source or surface | Linux 6.6.68 homelab capability set | Boundary |
| --- | --- | --- |
| Exec, network, DNS, cleartext HTTP, and protocol Aya sources | `BPF`, `PERFMON` | Runtime proven one source at a time on both homelab nodes. |
| Go TLS and fully symbolized cross-UID CPU profiles | `BPF`, `PERFMON`, `SYS_PTRACE` | Go 1.26.4 and a non-root Python stack were proven. Other TLS target permission layouts remain conditional. |
| Host resource source | none | Node CPU, node memory, and process observations were runtime proven with an empty effective capability set. |
| Processors, generators, and sinks | none beyond their source | These userspace stages do not independently load BPF or inspect another process. |

The opt-in `values-reduced-privilege.yaml` profile drops all inherited
capabilities and adds only `BPF`, `PERFMON`, and `SYS_PTRACE`. It keeps UID 0,
`RuntimeDefault` seccomp, no privilege escalation, and a read-only root
filesystem. `SYS_ADMIN`, `NET_ADMIN`, `NET_RAW`, `SYS_RESOURCE`, `SYSLOG`,
`CHECKPOINT_RESTORE`, and `DAC_READ_SEARCH` were absent from the proof. This is
a Linux 6.6.68 capability claim, not a rootless or universal-kernel claim. See
[ADR 0012](adr/0012-reduced-privilege-modern-kernels.md).

### Processing And Attribution Capabilities

| Processor | Default | Capability |
| --- | --- | --- |
| `processor.container_attribution` | Enabled | Enriches exec, process exit, network, DNS, request, trace, profile, dependency, flow, process-resource, and cgroup-resource signals with container and Kubernetes context when evidence exists. Attribution can use pid, cgroup id/path, container id, optionally cluster-wide Kubernetes Pod metadata, selected labels, node name, Pod IP, controller owner, Service ClusterIP, ready EndpointSlice fallback, and generic namespace/node/pod-label selectors. Pod identity wins over Service identity; a load-balanced ClusterIP is never guessed to be one backend Pod. Missing context remains missing rather than invented. |

### Capture Filter Capabilities

| Control | Default | Capability |
| --- | --- | --- |
| `[capture_filter]` | Disabled (no-op; every workload probed) | Scopes which Kubernetes workloads the eBPF sources *probe*, not merely enrich, with namespace exact/glob rules; label equality, inequality, existence, non-existence, set membership, and OR groups; exclude-wins OR groups; bounded executable/comm exclusions derived from container cgroups; and an explicit unknown-cgroup posture. The policy can express Guara's `proj-* AND guara.cloud/tier IN paid tiers AND guara.cloud/catalog-slug DOES NOT EXIST` contract directly. One process-wide controller owns the bounded Pod/Service/EndpointSlice snapshot for filtering and attribution; cluster-wide mode retains local Pods first, and the attribution processor never performs a second Kubernetes API request in the production CLI. Policy is evaluated in userspace; only cgroup ids and a posture byte cross into the kernel, where one bounded `HashMap` lookup per capturable handler decides capture. Filtering at connection establishment means an excluded workload's connections are never tracked, so its whole downstream read/write path early-exits, which reduces per-syscall overhead instead of only controlling scope. A bounded startup probe accepts only a directly mounted unified cgroup v2 root. Legacy, hybrid, and unavailable layouts override even an allow posture to deny before any Aya program attaches, with fixed hierarchy, compatibility, and fail-closed metrics. A new pod's UID is resolved directly from its cgroup path to shrink the bootstrap window. Suppressed handler invocations are counted into a per-source drop counter and reported, alongside allowed/denied cgroup counts, in a periodic accounting log, never silently. Softirq TCP-stat observations are node-scoped and never cgroup-filtered. |

### Sink And Export Capabilities

## Reading The Evidence Level

- **Implemented:** code path exists and is registered/configurable.
- **Local proven:** tests, fixtures, synthetic CLI, Docker smoke, or render checks
  prove the userspace behavior.
- **Runtime proven:** a guarded Linux/Kubernetes run recorded the claimed output.
- **Partial:** a useful slice is proven, but nearby production behavior remains
  outside the claim.

Detailed evidence lives in [proof-report.md](proof-report.md). Non-claims live
in [boundaries.md](boundaries.md).
